JS MODULES — WHAT EACH FILE SHOULD DO
api.js

Fetches from Met Museum API (no key) and Art Institute of Chicago API (no key)
Both return different JSON shapes — normalize into one consistent object:

js{
  title: string,
  artist: string,
  department: string,
  medium: string,
  artwork_date: string,
  image_url: string,
  source: 'met' | 'aic',
  source_id: string        // e.g. "met-436535"
}

Met API: https://collectionapi.metmuseum.org/public/collection/v1/
AIC API: https://api.artic.edu/api/v1/artworks — IIIF image server for images
Only artworks that have images should be included
Exports: fetchFeatured(), searchArtworks(query), fetchByDepartment(dept)

cache.js
Checks Supabase (db.from('artworks')) before hitting museum APIs
On cache hit → return cached rows
On cache miss → call api.js, insert results into Supabase, return data
Check cache using source_id to avoid duplicates
Also handles writing dominant_color back after Color Thief extracts it
Exports: getCachedArtworks(), searchCached(query), updateDominantColor(id, hex)
All queries use db from supabase.js (schema-scoped), NOT supabase

masonry.js
Takes an array of normalized artwork objects
Renders .art-card HTML into #masonryGrid
Removes skeleton cards once real data is ready
Initializes Masonry from masonry-layout package on the grid
Calls observeCards() from script.js after rendering
Art card HTML structure:

html<div class="art-card" data-id="123" data-source="met">
  <div class="art-card__img-wrap">
    <img src="..." alt="..." class="art-card__img" loading="lazy" />
    <div class="art-card__overlay"><i data-lucide="expand"></i></div>
  </div>
  <div class="art-card__info">
    <h3 class="art-card__title">Title</h3>
    <p class="art-card__artist">Artist Name</p>
  </div>
</div>

Uses event delegation on #masonryGrid for card clicks (not individual listeners)
Exports: renderGrid(artworks), reinitMasonry()

modal.js
Listens for card clicks via event delegation
Reads metadata from data- attributes on the clicked card
Fetches full artwork details from cache if needed
Runs Color Thief on the painting image to get dominant RGB
Darkens it: rgb(r * 0.35, g * 0.35, b * 0.35) — Spotify-style
Applies as gradient background to modal: linear-gradient(to bottom, darkened, #121212)
Modal shows: large image left, metadata right (title, artist, date, medium, department), "More from this artist" horizontal scroll below
Calls updateDominantColor() from cache.js to persist the color
Exports: openModal(artworkData), closeModal()

router.js
Simple History API router (no hash routing — looks unprofessional)
Maps URL paths to render functions
Used when artist pages or category pages are added later
Not needed for the single-page MVP — scaffold it but leave it minimal

Font paths in CSS are /assets/fonts/ (Vite serves public/ at root, so no /public/ prefix)
